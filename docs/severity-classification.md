# Severity Classification

CSA uses the severity classification built into GitHub's security advisory system. Advisories are rated as Critical, High, Medium, or Low.

## Severity Ratings

| Rating | Description |
|---|---|
| Critical | Exploitation is straightforward and causes severe impact |
| High | Exploitation is possible and causes significant impact |
| Medium | Exploitation requires specific conditions or causes limited impact |
| Low | Exploitation is difficult or impact is minimal |

## How Severity is Used

Severity classification informs:

- **Priority of response**: Higher severity vulnerabilities are addressed first
- **Disclosure urgency**: Critical and high severity issues are published and fixed as quickly as possible
- **Advisory content**: The severity rating is included in every published advisory

## Context Matters

Severity ratings provide a baseline, but CSA will consider the specific context of each vulnerability when prioritizing response. A medium-severity vulnerability in a widely-used CSA product may be prioritized over a high-severity vulnerability in an internal tool with no external users.
